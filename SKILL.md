# personal-deep-profile · 个人全维度探索档案生成器

> 沉淀自真实个人探索会话（49 张测测App截图 + 4 份 apesk 测评实测验证）。
> 比对话式 prompt 强在哪：多 agent 并行逐字转录不丢数据、出生信息多源交叉验证防排错盘、上升点独立天文核算、验收自循环保证 4 份产物齐活 + memory 永久沉淀（下次问流年零成本召回）。

## 🎯 目标

用户调用本 skill 后，预期达到的终态：

- 盘面数据已落为参数底稿且经过核验：转录模式=全部截图逐字转录**零遗漏**；自排盘模式=各体系**双源对算通过**（未过的体系砍掉并注明）
- 出生数据经 ≥2 处独立来源交叉验证一致（有出生地坐标时，上升点另经天文公式独立核算到角分级）
- 5 份档案文件已生成并交付：`个人命理综合分析报告.md`、`命理行动规划_*.md`、`命盘参数底稿_*.md`、`通俗版人物总结与行动指南.md`、（有测评时）`心理测评x命理交叉分析报告.md`
- 盘主档案已沉淀进 memory 并建索引——后续任何会话问流年/合盘/择日可直接召回，无需重发截图
- 中间产物（转码 JPG、/tmp 抓取文件）已清理；原始截图是否删除由用户拍板

## ✅ 验收清单

skill 在以下全部条件满足前**不得返回**：

- [ ] 盘面数据完整：转录模式 `grep -c '^===== IMG' <底稿>` == 图片数；自排盘模式 底稿中每个体系标注「双源核验：通过」或「已砍掉：原因」
- [ ] 出生信息一致性：≥2 处独立来源互不矛盾（转录=各图 birth_info；自排盘=用户输入 vs 反推节气/干支自洽。不一致必须停下问用户，不得擅自取舍）
- [ ] 产物齐全且非空：`for f in <5份文件>; do test -s "$f" || echo MISSING:$f; done` 无输出（无测评时为 4 份）
- [ ] 通俗版零术语：`grep -cE "用神|食伤|化忌|宫位|Cazimi|刑冲|大限|流年|主导功能" 通俗版*.md` 为 0（出现任一命理/心理学术语即不合格，回去重写）
- [ ] 每份报告含免责声明：`grep -l 'AI 生成' <各报告>` 命中全部
- [ ] 中间产物已清理：`test ! -d <输入目录>/converted && ls /tmp/apesk* 2>/dev/null | wc -l` 为 0
- [ ] memory 已沉淀：盘主档案文件存在，且 `grep <档案名> <memory目录>/MEMORY.md` 命中

若 3 轮自循环后仍未全部通过，写"未达成"清单到日志，标 [TODO]，再返回。

## 📥 输入

**最小输入 = 一句话出生信息**：`日期 + 时间 + 地点 + 性别`（例：1995-08-20 14:30 杭州 女）。排盘由本 skill 自行计算，不依赖任何 App。

| 参数 | 必填 | 默认 | 用途 |
|---|---|---|---|
| `birth_info` | **二选一** | — | 出生日期时间（精确到分）+ 地点（城市即可）+ 性别 → 走【自排盘模式】 |
| `screenshots_dir` | **二选一** | — | 已有排盘截图的文件夹（测测/问真等 App 截图，HEIC/JPG/PNG）→ 走【转录模式】，借 App 排盘引擎背书 |
| `assessments` | 可选 | 无报告时**主动问一句** | 心理测评，三级降级：①报告 URL（最佳，含子维度强度）→ ②口述结果（如「INFP / 霍兰德 RSA / 盖洛普前5」，可用但标注【口述低保真】）→ ③都没有则跳过交叉模块，并提示可后补 |
| `subject` | 可选 | 问用户 | 盘主称呼 + 与用户关系（决定 memory 沉淀方式：本人→user 型档案；他人→独立档案） |
| `output_dir` | 可选 | screenshots_dir 或 `~/Downloads/personal-profile-<subject>/` | 报告输出目录 |

两者都给时：自排盘 × 截图互为校验（最高置信）。出生时间未知或只到时辰：上升/宫位类结论必须降级标注「时辰级精度」。

### 自排盘模式的计算引擎（首次运行自动安装）

| 体系 | 工具 | 核验方式 |
|---|---|---|
| 八字四柱/大运/神煞 | `pip install sxtwl`（寿星天文历，节气精确） | 与 `lunar_python` 双库对算，干支不一致即停 |
| 西洋占星行星/宫位 | `pip install pyswisseph`（瑞士星历表） | 上升/MC 另用独立天文公式手算核对（误差 >0.1° 停） |
| 紫微斗数 | `pip install py-iztro` | 四化与生年干口诀核对 |
| 灵数/玛雅 Kin | 纯算术（生日数字 / JD mod 260） | 公式内置，双算 |
| 人类图 | pyswisseph + 太阳回退 88° 设计盘 | 类型/权威与定义中心逻辑互检 |
| 七政四余/二十八宿 | 无可靠开源库 → 仅在【转录模式】提供 | 自排盘模式下跳过并在报告中注明 |

**CRITICAL：自排盘的第一风险是排错盘**——任何一个体系未通过双源核验，宁可在报告中砍掉该体系，也不得输出未经核验的盘面。

## 🔁 工作流

1. **理解目标**：复述输入参数 + 验收清单到会话（让用户看到听懂了）；判定走自排盘 / 转录 / 双模式。
2. **取得盘面数据**（按模式分支）：
   - 【自排盘】安装计算引擎 → 城市名转经纬度 → 逐体系排盘并**双源核验**（见输入章节核验表），产出与转录模式同构的「参数底稿」
   - 【转录】`sips -s format jpeg -Z 1800` HEIC 批量转 JPG 到 `<dir>/converted/`，统计图片数 N → 每 3 张图一个子 agent 并行**逐字转录**（行星度数宫位、紫微十二宫星曜亮度四化、八字四柱藏干神煞大运、相位矩阵……），结构化 schema 收集 `{file, chart_type, birth_info, content}`，禁止概括、禁止解读
3. **交叉验证**：
   - 【转录】汇总各图 birth_info 检查一致性；法达起点、小限起年、灵数先天数做旁证；**不一致 → 停，列矛盾问用户**
   - 【自排盘】双库/双公式对算全部通过才放行；未过的体系整体砍掉并在报告注明
   - 【双模式】自排盘 vs 截图逐项比对，差异即停问用户
4. **上升精度核算**：有出生地坐标时，用独立天文公式计算上升/天顶与盘面值比对（转录模式误差 >0.5°、自排盘模式 >0.1° 必须标注警告）。
5. **产出四件套**：
   - 《命盘参数底稿》：全部转录原文，按图归档
   - 《综合分析报告》：①参数总整理 ②跨体系共振（**只把 ≥3 体系互证的结论写为高置信**，单体系结论须标注）③分体系深度解读 ④大运流年时间线（含过去验证点）⑤分领域指南 ⑥开运要诀
   - 《行动规划》：当年逐月行动表（流月干支 + 行运叠加）+ 未来 2–3 年关键窗口 + 复盘节点；若 memory 中有用户成长计划，须对齐
   - 《通俗版人物总结与行动指南》：**全文零术语**（机器自检见验收清单），写给盘主的家人朋友也能看懂的版本。固定结构：①你是个什么样的人（3–4 段大白话）②你的天赋（哪里顺）③你的软肋（哪里堵）④你和人相处的样子 ⑤适合你的活法 ⑥未来两三年怎么走（按年份给行动）⑦需要注意的坑 ⑧一句话总结（可贴墙）。语气标准：像了解你二十年的长辈朋友在说话，不像报告
6. **测评交叉**（按 `assessments` 降级分支）：
   - 【URL】`curl` 抓取 → `iconv` GB18030→UTF-8 → 去 HTML 提取正文（**子维度强度数据是交叉发现的核心，必须完整提取**，如"纠结88%""某子维度100%背离"）
   - 【口述】用户没报告时主动问一句："有 MBTI/霍兰德/盖洛普等测评结果吗？类型口述也行"——口述结果可用，但交叉报告中标注【口述低保真：缺子维度强度，结论降一级】，并提示补测能解锁什么（例：只知 INFP 看不出"纠结 88%"这种与用神精确对撞的关键数据）
   - 【都没有】跳过本模块，报告附录留一行"测评交叉待补"
   - 产出《测评x命理交叉分析报告》：逐条交叉验证表 + 矛盾校正（测评与命理冲突处，以测评实证为准修订命理建议）
7. **沉淀**：写 memory 档案（出生数据 + 核心参数 + 关键结论 + 产物路径），更新 MEMORY.md 索引；盘主为用户本人时，同步用户中央档案指针（如有）。
8. **自检**：逐条跑验收清单的 bash 命令。
9. **修复循环**：未通过项 → 回到对应步骤重做；≥3 轮未过 → 记日志标 [TODO] 返回。
10. **收尾**：`rm -rf converted/ /tmp/apesk*`；SendUserFile 交付报告；**询问**用户是否删除原始截图（删除前必须确认，底稿即唯一存档需明示）。

## ⚙️ 副作用

- 写入路径：`<output_dir>/*.md`（4 份报告）；当前项目 memory 目录（档案 + MEMORY.md）；盘主为用户本人时追加用户中央档案指针（如有）
- 网络调用：仅 `assessment_urls` 指定的公开 URL（curl GET，无登录态）；联网遵循 web-access skill 规则
- 使用凭证：无（不需要任何 API key）
- 破坏性操作：**有**——①删除自建的 `converted/` 与 `/tmp/apesk*` 中间产物（无需确认）②删除用户原始截图（**必须用户明确确认后才执行**）
- 子 agent 消耗：约 N/3 个并行转录 agent（49 张图 ≈ 17 个 agent ≈ 90 万 subagent tokens），调用前向用户提示量级

## 🚨 失败回执

- 错误日志写到：`~/Library/Logs/personal-deep-profile/YYYY-MM-DD.log`（含失败步骤、未过验收项、已完成产物清单）
- 会话内**显式报告**失败原因与已完成部分（本 skill 为交互式调用，用户在场；禁止静默退出、禁止假装完成）
- 出生信息矛盾、转录覆盖率不足等数据问题：停下问用户，不猜
- 退出码语义：成功 0 / 崩溃 1 / 验收未达成（已标 TODO）2

## Troubleshooting

| 症状 | 原因 | 解法 |
|---|---|---|
| 转录数 < 图片数 | 子 agent 超时/挂掉返回 null | filter(Boolean) 后找出缺的文件名，单独补转录 |
| 出生信息各图矛盾 | 用户混入了别人的盘 | 列出矛盾清单问用户，按盘主分组重跑 |
| apesk 页面乱码 | GB2312 编码 | 先 `iconv -f GB18030 -t UTF-8 -c` 再解析 |
| 上升点与 App 差 >0.5° | 出生地/时区录入错误 | 在报告中标注警告，提示用户核对 App 内出生地设置 |
